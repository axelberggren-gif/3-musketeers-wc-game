# lib/supabase/ — Supabase clients + generated types

## Purpose
Three flavours of Supabase client (browser, server-with-cookies, service-role) plus the
Next.js middleware session refresher and the generated DB type definitions. Everything
that touches Supabase imports from here.

## Key files
- `client.ts` — `supabaseBrowser()`: anon-key client for client components.
- `server.ts` — `supabaseServer()`: anon-key client wired to Next.js cookies (RLS-aware,
  acts as the logged-in user). Also `supabaseService()`: service-role client (bypasses
  RLS, for cron and admin work only).
- `middleware.ts` — `updateSession()`: refreshes the Supabase session on every request,
  invoked from the root middleware.
- `types.ts` — generated by `supabase gen types typescript` (run `npm run db:types`).

## Conventions
- **Choose the right client**:
  - Client component / browser code → `supabaseBrowser()`.
  - Server component / server action / route handler acting on behalf of a user →
    `supabaseServer()`.
  - Cron / admin / migration backfills that legitimately need RLS-bypass →
    `supabaseService()`.
- Type-safe queries: `supabase.from("table").select(...)` infers from `types.ts`. If
  the DB schema changes, run `npm run db:types` to regenerate.

## Invariants (do not break)
- **Never call `supabaseService()` from user-facing code** without explicit
  authorization checks first. It bypasses RLS — leaks are catastrophic for a
  predictions game (you'd reveal locked picks).
- `supabaseServer()` is `async` (returns a Promise) because Next.js 16 `cookies()` is
  async. Always `await` it.
- When the DB schema changes (new migration), regenerate `types.ts` via
  `npm run db:types`. Commit the regenerated file in the same PR.

## Known gotchas
- The `cookies.setAll()` callback in `supabaseServer()` is intentionally wrapped in
  try/catch — Server Components cannot mutate cookies; ignoring the error there is
  correct (the middleware does the actual writing).
- Environment variables required: `NEXT_PUBLIC_SUPABASE_URL`,
  `NEXT_PUBLIC_SUPABASE_ANON_KEY`, `SUPABASE_SERVICE_ROLE_KEY` (server-only).
