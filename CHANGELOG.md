- 2026-06-09 (`pending`) Removed the upper limit on the **Total goals** outright prop guess (was capped 0–300). WC 2026 has 104 matches, so a high-scoring tournament (~3 goals/match) already clears 300 — the cap was a binding limit on a legitimate guess, and the closest-guess scoring (`score_total_goals_guess`) means an unbounded value can never inflate anyone's points (a wild number just loses). The cap lived in three places, all relaxed to a non-negative-only floor: `components/predict/NumberInput.tsx` (`max` is now **optional** — the input drops its `max` attribute and shows a `0+` placeholder when omitted; the other numeric props pass `max` and are unchanged), the `setTotalGoalsGuess` server action in `lib/predictions/actions.ts` (drops the `> 300` branch, keeps the integer-≥-0 check), and the DB CHECK constraint via migration `0025_relax_total_goals_cap.sql` (drop-then-add `total_goals_guess >= 0` — idempotent, append-only, no column-type change so no `npm run db:types`). The other numeric props (highest-match, Final goals, biggest margin, golden-boot tally, total red cards) keep their caps. Validation tests added in `lib/predictions/actions.test.ts`.
